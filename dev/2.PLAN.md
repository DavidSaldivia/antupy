# antupy Roadmap and Collaboration Protocol (Independent Plan)

## Planning Principles

- Prioritize trust and contributor throughput before major feature growth.
- Keep pre-1.0 flexibility, but enforce quality gates.
- Prefer small merges with explicit tests/docs over large refactors.
- Low-hanging fruit first: version/docs consistency, contributor contract, CI.

## Working Model

This plan is organized by **streams** and **phases**:

- Phase 0: immediate foundation (days)
- Phase 1: stabilization (weeks)
- Phase 2: expansion (1-3 months)

Each task has a priority tag:

- `P0` must happen first
- `P1` next value layer
- `P2` strategic/longer horizon

## Collaboration Protocol for New Developers

## 1) Branch and PR model

- Branch naming:
  - `feat/<short-topic>`
  - `fix/<short-topic>`
  - `docs/<short-topic>`
  - `refactor/<short-topic>`
- Keep PRs small (target 200-400 net lines when possible).
- One conceptual change per PR.

## 2) Definition of Ready (before coding)

Contributor must have:

- issue or task with acceptance criteria
- affected modules identified
- test strategy stated
- docs impact stated

## 3) Definition of Done (before merge)

Required for every PR:

- tests added/updated
- type/lint/test gates pass
- changelog updated if user-facing
- docs updated if behavior/API changed
- reviewer checklist completed

## 4) Reviewer checklist

- correctness and domain validity
- unit consistency and dimensional safety
- backward compatibility intent explicit
- tests cover success + failure paths
- naming and docs clarity

## 5) Merge policy

- no direct pushes to main
- at least one approving review for non-trivial changes
- squash merge with conventional summary line

## 6) Release protocol

For each release:

1. Freeze scope.
2. Ensure changelog completeness.
3. Validate docs version string against package version.
4. Run release checklist.
5. Tag and publish.
6. Post-release smoke install and import test.

## 7) Communication cadence

- Weekly async update in one file/thread:
  - done
  - next
  - blockers
- Monthly roadmap review and reprioritization.
